# Linking Checklist

Use this reference when auditing or improving internal linking.

## Core Paths

- brand pages should expose relevant models
- model pages should expose main recambio categories and problems
- problem pages should point to the most relevant recambio category or SKU block
- category pages should help users jump to nearby problems or model hubs when useful

## Orphan Risk Signals

- generated pages with no contextual inbound links
- problem pages only reachable from sitemap or direct URL
- category pages missing from the model hub
- model pages not surfaced from brand or vertical pages

## Good Anchor Examples

- `batería para Xiaomi M365`
- `filtro compatible para Roomba i7`
- `qué hacer si tu Dyson V11 no carga`
- `pastillas de freno para Ninebot Max G30`

## Bad Anchor Examples

- `ver aquí`
- `más info`
- `producto`
- `haz clic`
