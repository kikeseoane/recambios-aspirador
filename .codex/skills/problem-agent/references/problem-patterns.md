# Problem Patterns

Use this reference to keep problem pages useful, distinct, and connected to recambios only when justified.

## Good Problem Page Pattern

- clear symptom in the title
- 2-4 likely causes
- 3-5 checks ordered from easiest to most informative
- a short fix hint or solution path
- CTA only when a part replacement is a realistic next step

## Weak Problem Page Signals

- generic advice that could fit any appliance
- no link between symptom and recambio category
- repeated causes across many pages without differentiation
- title variation only, same intent underneath

## Anti-Duplication Heuristic

Before adding a new problem page, check whether an existing page already covers:

- same main symptom
- same root cause family
- same CTA target

If yes, expand the existing page instead of creating another thin variant.
